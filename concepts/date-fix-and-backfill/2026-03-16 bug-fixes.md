# Bug Fixes: Wrong Game Date and Missing Backfill

Date: 2026-03-16

## Bug 1: Wrong date for late-night games

### Problem

`processGame()` in `src/pipeline.ts` received a `gameDate` parameter derived from the schedule API's `game.gameDate` ISO timestamp. For late US games, the UTC date rolls to the next day, so a game played on 2026-03-15 in Pacific time could be stored as 2026-03-16.

### Fix

`processGame()` now reads `liveFeed.gameData.datetime.officialDate` from the live feed response (which it already fetches). This field contains the correct local game date as YYYY-MM-DD. The passed-in `gameDate` parameter is kept as a fallback for the rare case where `datetime` is missing from the response.

### Changed file

- `src/pipeline.ts` line 58: added `officialDate` extraction, passed to `detectOutfieldAssists` instead of raw `gameDate`

### Type support

The `LiveFeedResponse` type in `src/types/mlb-api.ts` already had `gameData.datetime?.officialDate` defined, so no type changes were needed.

## Bug 2: No backfill on startup

### Problem

`startScheduler()` in `src/daemon/scheduler.ts` only looked at today's games. If the daemon was down for a day (container restart, deployment, etc.), games from missed days were silently lost.

### Fix

Added `backfillMissedDays()` function that runs once at startup, before the main while loop:

1. Queries `SELECT MAX(date) as lastDate FROM plays` to find the most recent stored play date
2. If no plays exist, skips backfill (nothing to catch up from, could be a fresh deploy)
3. If lastDate is yesterday or today, skips backfill (nothing missed)
4. For each day from lastDate+1 through yesterday, calls `scanDate()` and `insertPlays()` for any results
5. Capped at 7 days maximum to avoid hammering the API

The db query is wrapped in try/catch because existing tests pass `{} as Database` (no real SQLite instance). The backfill gracefully skips if the db is not available.

### Changed file

- `src/daemon/scheduler.ts`:
  - Added import of `scanDate` from `../pipeline`
  - Added `MAX_BACKFILL_DAYS` constant (7)
  - Added `backfillMissedDays()` async function
  - Added call to `backfillMissedDays(options)` after the "scheduler starting" log, before the while loop

## Bug 3: Bad record in database

The Aaron Judge play (id=19, game_pk=788096) was stored with date `2026-03-16` but should be `2026-03-15`. This was NOT fixed in code. It needs a manual SQL update:

```sql
UPDATE plays SET date = '2026-03-15' WHERE id = 19 AND game_pk = 788096;
```

## Testing

All 102 existing tests pass after these changes. No new tests were written.
